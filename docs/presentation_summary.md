# PPT 思路整理

原始汇报文件：`docs/transformer_report.pptx`

## 研究背景

传统 RNN/LSTM/GRU 在长距离依赖建模和并行训练效率上存在局限。Transformer 使用自注意力、多头注意力、编码器-解码器结构、残差连接和层归一化提升序列建模能力，适合处理患者问题中长文本、口语化和语义复杂的场景。

## 数据集

CMID 是中文医疗意图识别数据集，包含原始用户提问、医疗实体、分词结果、4 类一级意图标签和 36 类二级细粒度意图标签。项目将其用于医疗问答系统的意图理解与查询标准化。

## 动机

CMID 原始数据只有标签，没有具体回答。患者问题通常包含错别字、口语化表达、冗余描述和多意图混合，仅做分类无法直接回答用户。因此项目引入查询重写和检索增强：

- 分类负责理解问题属于什么医疗需求。
- 查询重写负责把口语问题映射到标准医学检索空间。
- RAG 知识库负责补足具体回答。

## 模型框架

模型采用多任务 Transformer：

- Encoder：BERT/Transformer 编码原始问题，输出上下文向量。
- Classification Heads：并行输出 4 类与 36 类意图 logits。
- Decoder：通过 Cross-Attention 读取 Encoder 表征，生成标准检索 Query。
- Dynamic Loss：用可学习同方差不确定性参数动态平衡分类损失与生成损失。

## RAG 流程

1. 从 CMID 中抽取治疗方案/治疗方法相关问题。
2. 使用 LLM 生成谨慎、通用、非诊断性的医学回答，形成问答知识库。
3. 对知识库问题使用 `jieba` 分词并建立 TF-IDF 索引。
4. 用户问题经 Decoder 重写为标准 Query。
5. 计算重写 Query 与知识库问题的余弦相似度，返回 Top-K 答案。

## 汇报指标

| Metric | Value |
| --- | ---: |
| Accuracy (L4) | 72.02% |
| Accuracy (L36) | 48.04% |
| BLEU-4 | 0.2407 |
| RAG Recall@1 (Ours) | 28.00% |
| RAG Recall@1 (Raw) | 10.00% |

这些指标反映出查询重写相较原始查询能提升检索命中率，但医疗语义细粒度分类与生成式重写仍受数据噪声、类别不平衡和模板表达限制影响。
